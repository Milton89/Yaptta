Android Walkie Talkie

In this project the goal is to use the Android Phones as Walkie-Talkies (push to talk) devices. The idea is to have a group of users specified in some way that can talk to each other as follows: when one user activates its microphone (e.g., by pressing a button on his/her phone), the voice from that user is digitized, packetized and the packets from that user are forwarded to all the other users (as VoIP packets), where they are played on their phones. While one user talks, all other microphones are disabled, for conflict resolution, you can use an ID number or such.

The groups can be specified in a number of ways, any of them are OK:

  • The users can specify a channel (eventually with a password) and thus is joins the group.
  • The users can explicitly select the recipients of their packets (from a list).
  • The groups are pre-established and the user can join any one group (or multiple groups). Not all users in a pre-established group may be present at any one time.

You can assume that the users are on the same IP network (so you can discover other users via broadcast), or, better yet, you can use a centralized server for service discovery (where you connect to see who's available).

This should work at least over WiFi, although it would be nice if the VoIP packets would be parsimonious enough to fit through the EDGE network (although the delay may be a problem).

A team attempted the project in Fall 2009. They did a half-decent job, but used the wrong APIs for transfering the data, and thus it worked pretty poorly.
